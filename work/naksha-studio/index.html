01 / Idea

AI could write code.
It still needed design judgment.

Naksha began as an attempt to encode the parts of design practice that generic coding agents routinely miss: hierarchy, interaction states, accessibility, brand consistency, motion, and the reasoning that connects them.

02 / Architecture

Not one giant prompt.
A system of specialists.

A design manager routes work to the relevant roles and loads only the knowledge required by the task. The same underlying design practice is packaged for different tools instead of forcing every environment into one installation model.

Principle 01

Progressive context

Load specialist knowledge only when the current task needs it.

Principle 02

Tool-native packaging

Respect the conventions of Claude Code, Cursor, Windsurf, Gemini CLI, and Copilot.

Principle 03

Quality as behavior

Build accessibility and design review into workflows rather than leaving them as advice.

03 / Adoption
