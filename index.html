How I work

How I take a product
from idea to production.

01

Map the workflow

I identify the users, operators, model decisions, handoffs, and failure states before committing to an interface.

  • System mapping
  • User flows
  • Failure states
02

Prototype the risky parts

I test agent responses, intervention paths, and complex interactions early—before polish makes weak assumptions expensive.

  • Agent behavior
  • Prototyping
  • Human control
03

Stay through production

I build the interface, connect it to real system behavior, and iterate on the states that only appear after people start using it.

  • React & Next.js
  • AI integration
  • Production iteration
